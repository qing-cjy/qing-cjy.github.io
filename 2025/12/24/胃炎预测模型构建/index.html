GGERD分类项目复盘：数据预处理是模型性能的核心驱动力

在GERD（胃食管反流病）三分类机器学习项目中，我经历了从模型准确率0.4-0.6的低谷，到最终提升至0.81-0.94的突破。核心结论非常明确：90%的性能提升来自数据预处理，仅10%来自算法选择与调参。这篇复盘将分享项目中遇到的核心问题、系统化预处理方案及关键经验。

一、项目背景与核心问题

### 项目目标

基于311个样本、57个临床特征（含基本信息、体征、化验指标等），实现GERD三个亚型的分类，辅助临床诊断。

### 初始困境

采用常规数据清洗（基础缺失值填补、标准化）后，测试决策树、KNN、随机森林、XGBoost等8种常用模型，准确率均徘徊在0.4-0.6之间，远未达到临床应用要求。后续尝试解决类别不平衡（合并少数类为二分类）、降维（PCA）、超参数调优等常规方案，均未取得实质性突破。

### 核心问题定位

所有模型表现不佳的根源并非算法，而是数据质量问题——特征类型混杂未区分、缺失值处理粗糙、异常值未合理处理、缺乏有临床意义的特征工程。

二、系统化数据预处理方案

针对数据质量问题，我构建了可复现的预处理流水线，封装为GERDDataPreprocessor类，核心功能包括完整日志记录、自动化特征识别、持久化保存、一键运行。以下是核心处理步骤及代码片段：

1. 自动化特征识别与差异化处理

医学数据特征类型复杂，需先精准区分数值特征、分类特征、高基数分类特征，再针对性处理：

- 数值特征：根据分布特性处理缺失值（偏态分布用中位数，对称分布用均值），偏态特征用Box-Cox或分位数变换矫正

- 分类特征：基于医学关键词（舌、脉、苔、sex、smoke等）识别，用众数填补缺失值

- 高基数分类特征：采用目标编码避免维度爆炸，失败时回退标签编码

#### 核心代码（特征识别与缺失值处理）

class GERDDataPreprocessor:

  def __init__(self, random_state=42):

  self.random_state = random_state

  self.scaler = StandardScaler()

  self.label_encoders = {}

  self.onehot_encoders = {}

  self.preprocessing_log = [] # 日志记录

 

  def advanced_preprocessing(self, df):

  # 1. 自动化特征识别

  categorical_keywords = [‘舌’, ‘脉’, ‘苔’, ‘sex’, ‘smoke’, ‘drink’]

  self.numerical_features = df.select_dtypes(include=[‘int64’, ‘float64’]).columns.tolist()

  self.categorical_features = [col for col in df.columns if any(kw in col for kw in categorical_keywords)]

  self.high_cardinality_features = [col for col in self.categorical_features if df[col].nunique() > 50]

 

  # 2. 差异化缺失值处理

  # 数值特征

  for col in self.numerical_features:

  if df[col].isnull().any():

  fill_val = df[col].median() if df[col].skew() > 1 else df[col].mean()

  df[col].fillna(fill_val, inplace=True)

  self.preprocessing_log.append(f”{col}：用{fill_val:.2f}填补缺失值”)

  # 分类特征

  for col in self.categorical_features + self.high_cardinality_features:

  if df[col].isnull().any():

  mode_val = df[col].mode()[0] if not df[col].mode().empty else ‘Unknown’

  df[col].fillna(mode_val, inplace=True)

  self.preprocessing_log.append(f”{col}：用{mode_val}填补缺失值”)

  # 后续省略编码、变换等步骤

  return X, y

2. 异常值处理：保留临床意义的缩尾策略

医学数据中的异常值可能是患者真实病理状态，不可直接删除。采用IQR方法检测异常值后，通过缩尾处理限制数值范围，既避免异常值干扰模型，又保留临床信息：

#### 核心代码（异常值处理）

def handle_outliers(self, df):

  outlier_info = []

  for col in self.numerical_features:

  Q1 = df[col].quantile(0.25)

  Q3 = df[col].quantile(0.75)

  IQR = Q3 - Q1

  lower_bound = Q1 - 1.5 * IQR

  upper_bound = Q3 + 1.5 * IQR

  outliers = ((df[col] < lower_bound) | (df[col] > upper_bound)).sum()

  if outliers > 0:

  df[col] = np.clip(df[col], lower_bound, upper_bound)

  outlier_info.append(f”{col}：{outliers}个异常值缩尾处理”)

  self.preprocessing_log.extend(outlier_info)

  return df

3. 医学领域特征工程：提升特征解释性与预测力

结合医学原理构建复合特征，比纯数据挖掘特征更具临床意义，是性能提升的关键：

#### 核心代码（医学特征工程）

def medical_feature_engineering(self, df):

  # BMI（体重指数，关联GERD与肥胖关系）

  if ‘weight’ in df.columns and ‘height’ in df.columns:

  df[‘bmi’] = df[‘weight’] / ((df[‘height’] / 100) ** 2)

  # 血脂比值（代谢指标，关联炎症反应）

  if ‘total_cholesterol’ in df.columns and ‘high_density_lipoprotein_cholesterol’ in df.columns:

  df[‘cholesterol_ratio’] = df[‘total_cholesterol’] / (df[‘high_density_lipoprotein_cholesterol’] + 1e-10)

  # 肝功能比值（ALT/AST，判断肝脏受累情况）

  if ‘alanine_aminotransferase’ in df.columns and ‘aspartate_aminotransferase’ in df.columns:

  df[‘alt_ast_ratio’] = df[‘alanine_aminotransferase’] / (df[‘aspartate_aminotransferase’] + 1e-10)

  self.preprocessing_log.append(“完成医学复合特征构建：bmi、cholesterol_ratio、alt_ast_ratio”)

  return df

4. 类别不平衡处理：SMOTE+Tomek Links组合策略

拒绝简化问题（不合并三分类为二分类），采用SMOTE+Tomek Links组合方法：先通过SMOTE智能合成少数类样本，再通过Tomek Links清理类别边界模糊样本，避免过拟合的同时平衡类别分布：

#### 核心代码（不平衡处理）

def handle_imbalance_advanced(self, X, y):

  self.preprocessing_log.append(f”原始类别分布：{dict(Counter(y))}”)

  # SMOTE+Tomek Links组合处理

  smote_tomek = SMOTETomek(

  smote=SMOTE(sampling_strategy=’auto’, random_state=self.random_state, k_neighbors=3),

  tomek=TomekLinks(sampling_strategy=’auto’),

  random_state=self.random_state

  )

  X_resampled, y_resampled = smote_tomek.fit_resample(X, y)

  self.preprocessing_log.append(f”重采样后类别分布：{dict(Counter(y_resampled))}”)

  return X_resampled, y_resampled

5. 持久化与一键运行：保证可复现性

类中集成模型保存、数据导出、日志生成功能，支持一键运行完整流程，满足医学研究的可复现性与可审计性要求：

#### 核心代码（持久化与一键运行）

def save_preprocessed_data(self, X, y, output_dir):

  # 保存特征和标签

  os.makedirs(output_dir, exist_ok=True)

  X.to_csv(os.path.join(output_dir, “features.csv”), index=False)

  pd.DataFrame({‘type’: y}).to_csv(os.path.join(output_dir, “labels.csv”), index=False)

  # 保存编码器和标准化器

  joblib.dump(self.scaler, os.path.join(output_dir, “scaler.pkl”))

  with open(os.path.join(output_dir, “label_encoders.pkl”), ‘wb’) as f:

  pickle.dump(self.label_encoders, f)

  # 保存预处理日志

  with open(os.path.join(output_dir, “preprocessing_report.txt”), ‘w’) as f:

  f.write(“\n”.join(self.preprocessing_log))

 

  def run_complete_preprocessing(self, data_path, output_dir, balance_data=True):

  df = pd.read_csv(data_path)

  X, y = self.advanced_preprocessing(df)

  if balance_data:

  X, y = self.handle_imbalance_advanced(X, y)

  self.save_preprocessed_data(X, y, output_dir)

  return X, y

### 关键发现

- 简单的逻辑回归模型准确率提升至0.9065，说明预处理后的数据已接近线性可分，数据质量的提升是核心驱动力；

- 树模型（XGBoost、随机森林）因能捕捉特征间复杂交互，表现最优，更适配多因素影响的医学数据场景。

四、核心经验总结

1. 数据优先于算法

当所有模型都表现不佳时，应首先检查数据质量而非盲目追求复杂算法。本项目90%的性能提升来自预处理，印证了“垃圾进，垃圾出”的核心原则。

2. 领域知识是关键

基于医学原理构建的BMI、血脂比值等特征，比纯粹数据挖掘的特征更具解释性和预测力。数据科学家必须与领域专家深度协作，让特征具备业务/临床意义。

3. 系统化胜过临时方案

构建可复现的预处理流水线（如GERDDataPreprocessor类），比临时清洗脚本更具长期价值，尤其适配医学研究对可复现性和可审计性的严格要求。

4. 不要过早简化问题

初期尝试将三分类转为二分类虽能小幅提升指标，但掩盖了GERD不同亚型的临床意义（不同亚型治疗方案差异显著）。坚持解决原始问题，才能让模型具备实际应用价值。

五、实用建议（面向医学数据科学同行）

1. 数据深度诊断先行：建模前先检查数据分布、缺失模式、异常值、特征相关性，定位核心问题。

2. 构建系统化预处理流程：封装为可复用类，保留完整日志，保证可复现性。

3. 深度融合领域知识：与临床专家协作，创建有医学意义的特征，提升模型解释性与预测力。

4. 全面评估模型：不止关注准确率，还需分析混淆矩阵、AUC、召回率等指标（医学场景中，漏诊比误诊更严重）。

5. 坚持问题本质：不为短期提升指标而简化实际需求，确保模型贴合真实临床应用场景。

六、项目价值

本项目通过系统化数据预处理，使GERD分类模型具备了实际临床应用价值，同时验证了“高质量数据是有效模型基础”的核心原则。这套预处理方法论可迁移至糖尿病分型、肿瘤预后预测等其他医学数据分析任务，为同类项目提供可复用的解决方案。
